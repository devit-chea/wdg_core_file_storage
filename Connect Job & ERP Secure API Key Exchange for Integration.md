## Integration Technical Specification: PKCE-Backed API Handshake 

This document defines the technical architecture, database designs, and sequence flow for establishing a secure, bidirectional, and multi-tenant integration between the in-house ERP System and the Public Job Platform. 

## 1. Security Architecture Summary 

- Master Controller: The ERP System acts as the primary orchestrator. It generates the api keys, handles tenant assignment, and manages the revocation lifecycle. 

- Proof Key for Code Exchange (PKCE): Eliminates static shared .env secrets. Trust is established dynamically per-session. 

Separation of Secrets (Key-Pair Architecture): 

   - Key_Alpha (ERP Inbound): Generated by ERP → Stored on Job Platform → Used by Job 

   - Platform to write to ERP. Key_Beta (ERP Outbound): Generated by ERP → Stored on ERP → Used by ERP to write to 

   - Job Platform. 

- Storage Cryptography: * Outbound keys are stored using reversible encryption (AES-256-GCM) so they can be parsed into HTTP headers. 

   - Inbound validation keys are stored as one-way SHA-256 hashes to prevent system compromise if a database is breached.

## 4. REST API Endpoint Designs 

## Endpoint 1: The Token Exchange (Job Platform API) 

URL: POST https://api.jobplatform.com/v1/integration/exchange 

## JSON Request Body (From ERP Server): 

{ "temporary_code": "code_8ac93bfa09e3...", "state": "state_9103bc7e0a...", "erp_company_id": "erp_co_7718", "erp_outbound_key": "erp_key_alpha_secure_9238bc...", "code_challenge": "b579178129a3a938c823..." } 

## Verification Steps (Job Platform Engine): 

1. Query: Fetch handshake details from connector_handshakes where temporary_code matches request. 

2. Check Expiration: Validate expires_at > NOW() . If expired, delete the row and throw 400 Bad Request . 

3. Check CSRF State: Assert that the incoming state matches the database state . 

4. Assert Challenge Matching: Take the code_verifier stored securely in the user's browsersession cache. Run it through the SHA-256 generation function: 

## Generated Challenge = Base64URL(SHA256(code_verifier)) 

Validate that Generated Challenge is strictly equal to the code_challenge sent in the request body. 

5. Persist Keys: If all assertions pass, commit keys to connector_partner and delete the matching record from connector_handshakes . 

## Success JSON Response Body (To ERP Server): 

{ "status": "success", "message": "Integration linked successfully", "job_platform_org_id": "job_org_9930" }
